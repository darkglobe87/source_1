HOW TO ADD MOVIE ARTWORK
=========================

Every movie now shows artwork above its description - either a specific
drawable (for the 4 movies below) or an automatically generated abstract
"poster" (ProceduralMoviePoster.kt) for everything else, seeded by the movie's
title so the same movie always looks the same. Nothing here depicts real
characters or recreates any actual poster - it's all original/generated, to
stay clear of using copyrighted movie art.

Four movies have a specific hand-made drawable (img_shrek.xml,
img_lord_of_the_rings.xml, img_pulp_fiction.xml, img_jaws.xml, already in
app/src/main/res/drawable/) instead of the generated fallback. If you want to
replace any of these with real licensed artwork later, just overwrite the
matching filename below - the code looks it up by name, nothing else to change.

Movies expecting a specific file, and the EXACT filename each one needs:

    Shrek                    -> img_shrek.png
    The Lord of the Rings    -> img_lord_of_the_rings.png
    Pulp Fiction              -> img_pulp_fiction.png
    Jaws                      -> img_jaws.png

(.jpg or .webp also work - just keep the base name identical.)

WHERE TO PUT THEM:
    app/src/main/res/drawable/

RULES - same as the audio file, and this is the rule that broke the last
build, so it matters:
    Filenames in any res/ subfolder must be lowercase letters, digits, and
    underscores ONLY. No capitals, spaces, or hyphens. "img_shrek.png" is
    correct. "Img-Shrek.png" or "img shrek.png" will FAIL THE BUILD.

Do not put any other files (readmes, notes, etc.) inside res/drawable/ or
res/raw/ - anything in those folders gets validated as a resource name.
Keep notes like this one at the project root instead, like this file is.

The code looks up each image by name at runtime (not a compile-time
reference), so you can add these one at a time, in any order, and rebuild
whenever - nothing needs to change in the code itself.
